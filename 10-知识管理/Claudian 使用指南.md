# Claudian 深度能力手册

> 我不是问答机器人。我能读、能写、能搜、能审、能规划、能并行。**但前提是你给我高质量的输入。** 这个手册不只告诉你"我能做什么"——更告诉你"怎么问才能让我做到最好"。

---

## 一、在你看具体功能之前——先看这个

### 为什么有时候我显得"弱智"

| 你说的 | 我实际接收到的 | 结果 |
|---|---|---|
| "帮我看看数据治理" | 看什么？哪个文件？什么角度？多深？ | 我只能给泛泛的回复 |
| "整理一下" | 整理成什么格式？放哪里？保留什么删什么？ | 我可能猜错你的意图 |
| "这个对吗" | 对的标准是什么？跟什么对比？ | 我只能说"看起来没问题" |
| "给我搞个方案" | 给谁看的？什么场景？多长？什么风格？ | 搞出来的可能不是你想要的 |

**核心原则：你给我越具体的约束，我输出越精准的结果。模糊不是安全，模糊是浪费。**

### 5 秒诊断你的提示词

问自己四个问题：

```
1. 我给角色了吗？（"你是面试官" vs 没说）
2. 我给了具体目标了吗？（"找出3个逻辑矛盾" vs "帮我看看"）
3. 我给了输出格式了吗？（"逐条列出，标严重程度" vs 没说）
4. 我给了边界了吗？（"只基于我的笔记" vs 没说）
```

缺任何一项 → 补上再发。

### 不知道怎么开口时的决策路径

```
你有东西要处理
    │
    ├─ 是课程/书本笔记 → 跳到 [[#1.1]]
    ├─ 是想检查错误 → 跳到 [[#2.1]]，选一种审计
    ├─ 是准备面试 → 跳到 [[#3]]，选场景
    ├─ 是要串联知识 → 跳到 [[#5.1]]
    ├─ 是要写东西 → 跳到 [[#6]]，选输出类型
    ├─ 是项目相关工作 → 跳到 [[#4]]
    ├─ 是要查外部信息 → 跳到 [[#5.2]] / [[#5.3]]
    └─ 不太确定 → 直接描述情况，加一句"你先问我要澄清什么"
```

## 一、知识内化

### 1.1 三要素结构化

```
整理这份笔记，按"这是啥/有啥用/怎么用"三要素重构。
加 frontmatter（tags、alias、status），存到 [目标目录]。
只保留原文案例，不要自己编新案例。
```

⚠️ **容易踩的坑**：一次只扔一个主题的笔记。不要把三门课混在一起扔过来——我会把不同课程的内容混编，结构就乱了。

### 1.2 口语化翻译

```
把 [某概念/某章节] 翻译成面试口语版。
要求：像唠嗑一样自然、每段 60 秒能说完、带具体数字或场景。
```

⚠️ **注意**：原文没有的案例不要说"加个例子"——我会自己编。如果不让我自己编，加一句"只用原文案例"。

### 1.3 记忆闪卡

```
基于 [笔记/章节]，生成 [10-20] 对闪卡。
Q 用面试官口吻，A 口语化，3 句话内。
```

### 1.4 比喻串讲

```
把 [概念/流程] 用一个日常场景从头串到尾（装修/做饭/搬家/开车/挂号看病）。
一条主线到底，不要东一个比喻西一个比喻。
```

⚠️ **关键**：指定一个场景。不指定的话我可能选一个你觉得不贴切的。

### 1.5 实战清单

```
把 [知识点/课程] 变成可执行的时间线清单。
按天/周排列，每步写清楚：找谁、做什么、产出什么。
```

### 1.6 学习路径

```
我要学 [某领域]，已有这些资料：[列出]。
帮我设计学习路径：先学什么后学什么、哪些可以跳过、哪些必须精读。
```

## 二、知识审计

> ⚠️ **重要前提**：审计质量取决于对比标准。不说标准我只能"感觉"——说了标准我能逐条对标。

### 2.1 事实核查

```
审计 [文件/章节]，逐条检查：
- 日期、法律条款号、人名是否准确
- 技术定义是否与 [行业标准] 一致
每项标：🔴事实错误 / 🟡存疑需验证 / ✅确认无误
```

### 2.2 逻辑一致性审计

```
检查 [文件A] 和 [文件B] 之间有没有逻辑矛盾。
同一个概念在两个地方说法不同？分类体系互相冲突？
```

### 2.3 完整性对比

```
对照 [标准/体系，如 DAMA-DMBOK2、PMP、DCMM]，检查我笔记缺了什么知识域。
列出缺失项，按重要性排。
```

### 2.4 时效性审查

```
检查 [文件] 涉及的政策、法规、标准版本是否还有效。
列出已过时的项，并给出当前最新版本。
```

### 2.5 盲点扫描

```
我在学 [某领域]，目前覆盖了 [已学内容]。
这个领域还有哪些核心知识我没提到？哪些是面试/工作中高频出现但我没覆盖的？
```

## 三、面试全线

### 3.1 全真模拟

```
你是 [公司+岗位] 的面试官，风格 [温和追问型/压力测试型/技术深挖型]。
按真实节奏面试我。全部结束后逐条反馈。
```

⚠️ **关键**：指定风格很重要。温和追问型和压力测试型问法完全不同。

### 3.2 弱点诊断

```
假设我明天面试 [岗位]，这份资料里哪些地方最容易被质疑？
标严重程度：🔴致命 / 🟡需准备 / 🔵加分项可优化
```

### 3.3 五层追问

```
针对 [某个项目/知识点]，模拟面试官连问 5 层。每层比上一层深一个维度。
```

### 3.4 角色扮演

```
你是 [具体角色]，场景：[描述]。跟我对线，看我能不能搞定你。
```

### 3.5 JD拆解

```
这是 JD：[粘贴]。拆解：核心要求、我的匹配度、差距、面试中怎么弥补。
```

### 3.6 谈薪策略

```
拿到 [公司+岗位] offer，薪资范围 [X-Y]，我期望 [Z]。
给策略：什么时候开口、怎么开口、锚定多少、底线多少。
```

## 四、项目管理

### 4.1 风险分析

```
分析 [项目方案] 的风险点。
每个风险标注：概率、影响、触发信号、应对方案。
```

### 4.2 干系人策略

```
分析 [项目] 的关键干系人。
每个人的：立场、影响力、关注点、我怎么争取或管理。
```

### 4.3 项目复盘

```
基于 [项目资料/笔记]，做复盘报告。
结构：目标回顾→实际结果→偏差分析→根因→改进措施→可复用经验。
```

### 4.4 会议纪要

```
把这段会议笔记整理成纪要：决策事项、行动项（谁/做什么/何时）、遗留问题。
原文信息不全的地方标注出来让我补充。
```

## 五、信息搜集与串联

### 5.1 跨库串联

参见 [[10-知识管理/知识库管理守则]] 中的串联模板。或一句话：

```
把 [知识点A]（在 [文件A]）和 [知识点B]（在 [文件B]）串起来写一篇串联笔记。
```

### 5.2 网页查证

```
搜索 [某信息] 的最新状态，和我笔记里的对比，列出差异。
```

### 5.3 网页加工

```
把这个网页 [URL] 的核心内容提取出来，按三要素整理，存到 [目录]。
```

⚠️ **限制**：需要登录的网页（如飞书、企业内网）我抓不了。

### 5.4 图片/PDF分析

```
看我笔记里的这个架构图/截图/PDF，提取关键信息整理进笔记。
```

⚠️ **限制**：PDF 超过 20 页需要指定页码范围。图片清晰度影响识别准确率。

## 六、写作输出

### 6.1 邮件与函件

```
写一封 [场景] 邮件：给 [谁]，语气 [正式/轻松/催办]。
要点：[列出]。要求：[简短/有数据/有下一步]。
```

### 6.2 PPT与汇报

```
根据 [笔记/项目]，写 [X] 页 PPT 大纲。每页：标题 + 一句话核心 + 一个支撑数据。
```

### 6.3 简历与自我介绍

```
基于我的 [经验/项目列表]，打磨 [简历/自我介绍]。
挑出：太虚的、缺数字的、可以更有力的地方。
给 3 个版本：30秒/90秒/文字版。
```

## 七、决策与思考

### 7.1 决策辅助

```
我在纠结 [A] vs [B]。做利弊分析：优势/劣势/风险/隐性成本。给建议，附理由。
```

### 7.2 深度思考

```
用第一性原理拆解 [问题]：最基本的约束是什么？抛开现有假设，从零开始怎么解决？
```

### 7.3 反向推演

```
假设 [方案] 最终失败了，最可能的 5 个原因是什么？现在应该提前防什么？
```

### 7.4 头脑风暴

```
针对 [问题/目标]，给 10 个可能的解决方向。先不评判，只管发散。
```

## 八、库管理

### 8.1 批量文件操作

```
把 [目录] 里所有含 [关键词] 的文件移到 [目标目录]。
把所有 [旧标签] 批量替换为 [新标签]。
```

### 8.2 健康检查

```
检查 [目录/全库]：无 frontmatter 的文件、无标签的文件、孤立笔记、死链。
```

### 8.3 标签规范

```
扫描全库标签，列出：不规范的、同义异写的、建议统一方案。
```

### 8.4 MOC生成

```
扫描 [目录]，自动生成 MOC 索引页，按主题分类列出所有文件。
```

## 九、隐藏能力

### 9.1 定时任务

```
"每天下午 6 点提醒我检查 00-收件箱"
"每周一早上提醒我翻记忆闪卡"
```

⚠️ 只在对话空闲时触发，不是闹钟。持久化任务重启后还在。

### 9.2 并行Agent

复杂任务自动拆解并行。也可以主动触发：

```
"同时做三件事：审计A文件、整理B笔记、搜索C信息"
```

### 9.3 规划模式

```
"先别动手，出方案让我看"
```

每做完一步：
```
"每做完一个步骤，停下来让我确认再继续"
```

### 9.4 任务追踪

多步骤任务我会创建任务列表，标记进度。

### 9.5 多轮迭代

最好的结果很少一次到位。标准流程：

```
第一轮：整理笔记 →
第二轮："检查刚整理的有没有问题" →
第三轮："把金句精简到 5 条，每条加一个数字" →
第四轮："给每条金句配一个面试场景，什么时候用"
```

⚠️ **常见错误**：第一轮就要求完美。先快速出一版，再迭代打磨。

### 9.6 事实标注

```
"整理这份笔记，每个关键论断标注来源：我的笔记原文 / 行业标准 / 合理推断"
```

### 9.7 主动提问

```
"我不确定怎么描述这个需求，你先问我需要澄清什么"
```

## 十、避坑指南

### 坑1：一次扔太多

❌ 把三门课、五个项目经历混在一起扔给我
✅ 一次一个主题，处理完再下一个

### 坑2：不指定输出格式

❌ "帮我整理一下" ← 我只能猜
✅ "按三要素整理，加 frontmatter，存到 XX 目录，只保留原文案例" ← 精准

### 坑3：不告诉我目标场景

❌ "帮我看看数据安全"
✅ "我明天面试要讲数据安全，从面试视角检查我的笔记能不能撑住 3 分钟"

### 坑4：第一轮就追求完美

❌ 输入写很长很精确 → 期望一次性完美
✅ 先快速出一版 → 看结果 → 给反馈迭代 → 二三轮才是精品

### 坑5：不让我主动提问

❌ 含糊其辞地丢过来，也不说"不清楚的地方问我"
✅ "你先看，有不确定的问我" ← 我可以主动澄清

### 坑6：让我动手前没让我出方案

❌ 复杂任务直接说"做吧"
✅ "先出方案我看" ← 避免我走弯路，也避免你浪费时间

### 坑7：给我加工过的内容

❌ 自己先精简了一轮再给我
✅ 直接给我原始口水笔记——我能从你的语言风格里提取你自己说不出来的表达方式
