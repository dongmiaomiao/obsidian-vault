# Claudian 使用指南

> 我是运行在你 Obsidian 仓库里的 AI 助手。不是聊天机器人，是能读、能写、能搜、能审的知识库搭档。

---

## 一、我是什么

| 能做的 | 不能做的 |
|---|---|
| 读你 vault 里任何文件 | 访问 vault 之外的文件（除非你指定路径） |
| 创建、编辑、删除 .md 文件 | 处理视频、音频、图片 OCR |
| 全局搜索、按模式查找文件 | 浏览网页（受限） |
| 批量改名、合并拆分文件 | 执行 Obsidian 插件功能 |
| 审计知识内容（找错误、找矛盾） | 知道你今天做了什么（除非写在笔记里） |
| 按模板标准化笔记 | 主动提醒你（没有定时任务能力） |

---

## 二、核心工作流

### 1. 课程笔记加工（最常用）

**你说**：
> "帮我整理这份课程笔记，按'这是啥/有啥用/怎么用'三要素重构，带 frontmatter 和标签，存到 10-知识管理/数据治理课程笔记/"

**我会**：读你的原始笔记 → 理解结构 → 写成结构化笔记 → 写进 vault → 告诉你改了什么。

> ⚠️ 把视频内容变成文本是你的事（字幕导出、语音转文字），文本扔给我之后的一切是我的事。

### 2. 知识审计

**你说**：
> "检查我数据治理知识体系里有没有错误或矛盾"

**我会**：通读相关文件 → 交叉比对 → 列出问题（事实错误、逻辑矛盾、遗漏、过时信息）→ 给修正建议。

### 3. 批量重构

**你说**：
> "我现在 27 个文件太乱了，帮我合并精简"

**我会**：分析结构 → 设计新方案 → 说明理由 → 你确认后执行。

### 4. 跨库连接

**你说**：
> "数据治理知识体系里提到了 DCMM，跟中铝面试文档里讲的 DCMM 连起来"

**我会**：在两边文件里加 `[[wikilink]]`，形成双向链接。

### 5. 面试准备

**你说**：
> "根据我知识体系里的内容，模拟面试官问我数据治理项目经验"

**我会**：基于你的笔记出题 → 你回答 → 我挑刺（逻辑漏洞、表达不清晰、缺少数据支撑）。

## 三、高效提示公式

```
= 【角色】+【具体任务】+【输出格式】+【边界条件】

❌ "帮我看看数据治理"
✅ "假设你是面试官，针对我数据质量那节的笔记，出 5 个追问，每个追问标注它会暴露我什么弱点"
✅ "把这篇笔记按三要素重写，只保留原文里的案例，不要自己加新案例"
✅ "检查数据安全那节的法律时间线，和网上查到的有没有出入"
```

## 四、进阶用法

### 让我主动找问题，别等我被问

最浪费我的用法是"你看看这个对不对"——我会看，但不会主动告诉你哪里不对。

**更好的说法**：
> "假设我明天面试，这份知识体系有哪些地方会让面试官皱眉？给我列出来，标严重程度。"

### 让我串联，别让我单点回答

> "数据盘点的二八原则、主数据的双向同步、企业数据服务的库→文件→FTP，这三个知识点串成一条线讲给我听。"

### 让我扮演角色

> "你现在是甲方数据部门负责人，我刚入职两周，来找你拿数据库权限。考考我怎么说服你。"

## 五、我帮你避过的坑（举例）

| 坑 | 我发现的方式 |
|---|---|
| 数仓"无主键"一刀切 | 对比维度表和事实表的实际结构 |
| CDC 和数据策略放同一层级 | 分类逻辑审查 |
| 网络安全法时间写错 | 法律时间线交叉校验 |
| 比喻文件完全冗余（5 个文件重复内容） | 结构化审计 |
| 数据质量只讲 2 个维度缺 4 个 | 对照 DAMA 六维度标准 |
| 27 个文件散落互不链接 | 全库扫描 + 重建连接 |

---

## 六、最佳实践

1. **别让我猜**：告诉我你的目标（面试 / 入职 / 复习 / 写文档），我会针对目标调整输出
2. **扔原始材料，别扔加工过的**：你的口水笔记比"整理后的"对我更有价值——我能从中提取你自己的语言风格和案例
3. **让我审，别让我只写**：写完内容后加一句"检查有没有问题"
4. **一次给我一块完整的**：大课一次扔完，别分 5 次发，我能处理
5. **直接告诉我错了**：我做的比喻重复了、文件结构不合理了——直接说，我改
