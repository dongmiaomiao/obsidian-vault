## 一、知识内化

### 1.1 三要素结构化

**模板**：
```
整理这份笔记，按"这是啥/有啥用/怎么用"三要素重构。
加 frontmatter（tags、alias、status），存到 [目录]。
只保留原文案例，不要编。
```

**示例**——输入：
> 整理下面数据湖的笔记，三要素重构，存到 10-知识管理/数据治理课程笔记/

**示例**——我会输出一个文件 `数据治理_数据湖.md`：
```yaml
---
tags: [数据治理, 数据湖]
alias: [Data Lake, 数据湖概念]
status: reviewed
---
# 数据湖

## 这是啥
一种集中式存储，允许以原始格式存所有结构化/半结构化/非结构化数据，不要求先定义 Schema。

## 有啥用
解决"数据没地方存、格式不统一、分析时要到处找"的问题。
数据仓库需要先建模再入库，数据湖反过来——先入库、用到时再定义结构，适合探索性分析。

## 怎么用
1. 建湖：选 Hadoop/对象存储，确定分区策略
2. 入湖：原始数据直接灌入，不做ETL只做EL
3. 治理：做元数据目录，不然变数据沼泽
4. 消费：按需定义 Schema，对接分析引擎
```

### 1.4 比喻串讲

**示例**——输入：
> 把数据治理全流程用"装修老房子"串起来讲一遍

**示例**——我会回：
> "数据治理就是装修一套住了20年的老房子。第一步**数据盘点**：拿手电筒钻到地下看——哪些水管还在流水（在用系统）、哪些锈死了但还接在总阀上（停用但没下线的老系统）、哪些龙头压根不出水（空表）。第二步**数据标准**：全家统一水管口径，不能厨房用4分管、卫生间用6分管，接不上。第三步**主数据管理**：房主只能有一个名字——你在房产证、户口本、水电单上叫三个不同名字，谁知道这房到底是谁的？第四步**数据质量**：定期检查水压、水质，别等水管爆了才发现。第五步**数据安全**：总阀门加锁，不同房间不同钥匙，租客不能进主卧翻抽屉。"

### 1.5 实战清单

**示例**——输入：
> 把数据盘点知识变成入职新公司前两周的行动清单

**示例**——我会回：
```

第3-5天（筛系统）：跑SQL查每个表记录数和最后更新时间；标出"三无系统"（无业务方认领/无近期更新/无数据字典）→ 第一份裁撤建议清单。

第10-12天（定策略）：按二八原则砍掉80%的表，只留20张核心表；每张核心表标接入策略（全量覆盖 vs 增量追加）。

第13-14天（交答卷）：一页PPT告诉领导——"咱家有X个活系统，Y张核心表，Z个垃圾系统建议停掉。下周先接这3张最急的表。"
```

## 二、知识审计

### 2.1 事实核查

**示例**——输入：
> 审计数据安全那节，对照网络安全法和数据安全法逐条检查

**示例**——我会回：
```
🔴 网络安全法：笔记写"2017年施行"。实际上2016年11月7日通过，2017年6月1日施行。年份对但少了具体日期。

🔴 等保2.0：笔记写"2019年12月发布"。实际GB/T 22239-2019是2019年5月10日发布，2019年12月1日实施。混淆了发布时间和实施时间。

✅ 数据安全法：2021年6月10日通过，9月1日施行——正确。

🟡 建议补充：笔记没提到《个人信息保护法》（2021年11月1日施行），它和数据安全法是同一时期"三驾马车"之一，面试常考。
```

### 2.2 逻辑一致性审计

**示例**——输入：
> 检查知识体系里关于"主数据"的说法，和数据仓库那节的描述有没有矛盾

**示例**——我会回：
```
🟡 不一致：主数据那节说"主数据管理是双向同步，数据仓库是单向抽取"——正确。
但数据仓库那节讲ODS层时说"从主数据系统获取统一维度"——暗示MDM→DW是单向，跟MDM那节说的"双向"不矛盾，但没解释清楚"双向同步"指的是MDM和业务系统之间，不是和数仓之间。

建议：在数仓那节加一句"MDM与业务系统双向同步确保源头一致；数仓从MDM单向抽取维度数据，不反向写回MDM"。
```

### 2.3 盲点扫描

## 三、面试全线

### 3.1 全真模拟

**示例**——输入：
> 你是中铝数据治理PMO的面试官，温和追问型。基于我的项目经历面试我。

**示例**——我会这样开场：
> "董苗苗你好，看了你的简历，郑州智慧环保项目你统筹了9个团队90多人，这个体量在PMO岗位很有说服力。我想先听你聊一个具体场景——项目推进中，你发现某个部门提供的数据质量很差，但对方说'我们就这水平，你爱接不接'。你怎么处理？"

然后你回答，我追问，全部结束后逐条反馈：
- 你的"建立机制"说得太虚，需要一个具体操作:"我跟他们领导约了一次会，会上展示了他们数据的质量问题对下游报表的具体影响"
- 缺少数字支撑——这种场景最好带一个量化结果

### 3.2 弱点诊断

**示例**——输入：
> 假设明天面试中铝PMO，基于我的知识体系和简历，哪些地方最容易被质疑？

**示例**——我会回：
```
🔴 制造业经验缺失：这是最致命的问题。你的项目全是政务和医药。面试官一定会问"你懂铝业吗"。已经准备好了三明治答法（承认→桥接→行动）。

🔴 没有DAMA证书：JD上写"数据治理"但你没证。答法已备：正在学DAMA-DMBOK2+计划考CDGA。

🟡 简历上的"高质量数据集"具体指标：你说准确率99.8%，面试官可能追问"这个数字怎么算出来的、样本量多大、前后对比数据"——准备好计算口径。

🔵 中铝AI中台相关经验：你有AI辅助项目管理经验但简历上不明显。建议在自我介绍里加一句"近年将AI融入项目管理全流程"。

### 3.3 五层追问

**示例**——输入：
> 针对郑州智慧环保项目，五层深挖

**示例**——我会出：
```
第1层："这个项目具体是干什么的？你管了什么？"
第2层："90个人9个团队，你怎么分工的？遇到最严重的协调问题是什么？"
第3层："你说建了三级质量管控体系——具体是哪三级？每一级的指标是什么？谁来执行？"
第4层："如果重来一次，这个项目你会在哪个环节做不同的决策？"
第5层："你从这个项目沉淀了什么可复制的方法论，到一个完全陌生的行业也能用？"

→ 如果第3层你只能用"制定了标准、做了稽核"这种话搪塞，面试官就知道你没亲自执行过质量管控。
```

## 四、项目管理

### 4.1 风险分析

**示例**——输入：
> 假设我入职中铝PMO，第一次主导铝业数据治理项目。帮我做风险分析。

**示例**——我会回：
```
🔴 行业知识不足（概率高/影响高）
触发信号：开会听不懂铝业术语（电解铝/氧化铝/炭素/铝加工）、不了解生产流程
应对：第一周下到子公司生产线，跟业务人员走一遍完整数据流。找一本铝业入门书/行业报告快速建立知识框架。前两周不谈方案、不拍板，只做调研。

🔴 干系人抵触（概率高/影响中）
触发信号：子公司数据负责人推脱、以"安全"为由拒绝数据共享
应对：不先去"要求"共享数据，先让每个子公司讲自己的痛点。提"最小必要原则"——只共享业务必需的字段。技术上做适配层，不要求对方改造系统。

🟡 项目周期被压缩（概率中/影响高）
触发信号：领导层突然要求提前交付
应对：提前准备好"可分阶段交付"的方案——第一期先接3张核心表打通流程，第二期扩展覆盖。不要等压力来了再想办法。

🟡 数据质量比预期差（概率中/影响中）
触发信号：接入后发现大量空值、乱码、不一致
应对：在项目计划里预留20%工期做数据清洗。接数据前先做抽样质量评估，提前管理甲方预期。

🔵 AI中台对接复杂度（概率低/影响中）
触发信号：坤安大模型团队提出新的数据需求
应对：保持和AI中台团队的定期沟通，提前了解他们的数据需求，纳入盘点范围。
```

### 4.3 项目复盘

**示例**——输入：
> 基于郑州智慧环保项目，做一份复盘报告

**示例**——我会回（节选结构）：
```
# 郑州智慧环保项目复盘

## 目标回顾
- 整合6部门数据，构建4大主题库
- 建立三级质量管控体系，准确率目标99.5%
- 90天内完成平台上线

## 实际结果
- 4大主题库按计划建成
- 准确率做到99.8%，超出目标
- 交付延期12天（数据对接延迟导致）

## 偏差分析
延期根因：对某委办局的数据对接难度预估不足。他们的老系统（COBOL）没有API，需要开发中间件适配层。这个风险在项目初期没被识别。

## 改进措施
1. 项目启动时增加"系统技术栈摸排"，对老系统提前评估对接方案
2. WBS里给老系统对接预留30%缓冲时间
3. 提前准备备用方案（手动导入→半自动→全自动的渐进路径）

## 可复用经验
- 三级质量管控体系（规则核查→人工抽检→问题跟踪闭环）可直接复用到其他行业
- 反向模拟法查字段含义——适合所有"没文档+老系统"场景
- 用"最小必要原则"化解跨部门数据共享阻力
```

### 4.4 头脑风暴

**示例**——输入：

**示例**——我会回：
```
1. 数据视角："原计划接入8个系统，实际有13个在用，3个是没人承认的'影子系统'——不盘不知道，盘了才发现体量翻倍"
2. 质量视角："6个系统中4个存在严重质量问题，如果强行上线，后续返工成本会翻3倍"

## 七、决策与思考

### 7.1 决策辅助

**示例**——输入：
> 我在纠结：先考PMP还是先考CDGA数据治理认证。帮我做利弊分析。

**示例**——我会回：
```
选项A：先考CDGA
优势：你现在面的是数据治理PMO岗位，CDGA与岗位直接对口，面试时"正在备考CDGA"比"有PMP"更有说服力。CDGA考试相对简单（100道选择题），备考周期短（1-2个月）。
劣势：通用性不如PMP，纯项目管理岗位不认这门证。

选项B：先考PMP
优势：行业通用性最强，你已经有PM经验，备考周期约3个月。
劣势：你已经有PMP认证了（简历上写的"PMP认证"），不需要再考一次。等等——你简历写的"PMP认证"，如果你还没证书的话需要澄清。

建议：先CDGA。理由——你当前目标是数据治理PMO岗，CDGA直接加分。考完CDGA立刻更新简历。如果还没PMP证书，CDGA之后马上备考PMP。
```

### 7.2 第一性原理

**示例**——输入：
> 用第一性原理拆解：面试中怎么讲项目经验最有说服力

**示例**——我会回：
```
第一性原理：面试官不关心"你做了什么"，只关心"你能帮我解决什么问题"。

从零拆解：
1. 面试官的心理约束：他不知道你的项目是真的还是编的。他唯一能判断的依据是——你提到的细节是否具体、逻辑是否自洽、数字是否可验证。
2. 所以你要做的不是"介绍项目"，而是"让他感觉你在现场"——提到具体的系统名、具体的冲突、具体的数字变化。
3. 最优结构：场景→卡点→我的动作→量化结果→可复用方法论。每个1分钟。

推导出的实用规则：
- "我们做了数据治理" → 无效，太虚
- "打通6个部门数据，发现30%的字段为空，花了2周跟业务一个个对，最终确定了12个核心字段" → 有效，像在现场
```

## 九、隐藏能力

### 9.1 定时任务
> "每天下午6点提醒我检查00-收件箱"
⚠️ 对话空闲时触发，不是闹钟。

### 9.2 并行处理
复杂任务自动拆解并行。也可主动触发：
> "同时做三件事：审计A、整理B、搜索C"

### 9.3 规划模式
> "先别动手，出方案让我看"
> "每做完一步停下来让我确认"

### 9.4 多轮迭代
第一轮快速出 → 第二轮检查问题 → 第三轮精简 → 第四轮加场景。**别第一轮追求完美。**

### 9.5 事实标注
> "整理这份笔记，每个论断标注来源：原文/标准/推断"

### 9.6 主动提问
> "我不确定怎么描述需求，你先问我要澄清什么"

## 十、避坑指南

| 坑 | ❌ | ✅ |
|---|---|---|
| 一次扔太多 | 三门课混一起 | 一次一个主题 |
| 不指定格式 | "整理一下" | "三要素，加frontmatter，存XX目录" |
| 不给目标 | "看看数据安全" | "明天面试要讲数据安全，检查能否撑3分钟" |
| 追求一次性完美 | 写很长的输入期望一遍出精品 | 先快速出一版，再迭代 |
| 不让我问 | 含糊丢过来 | "有不确定的问我" |
| 不给方案 | 复杂任务直接"做" | "先出方案我看" |
| 给我加工过的 | 自己先精简了再给我 | 直接给原始口水笔记 |
