## 一、防止知识库变垃圾堆

你现在的 vault 已经有这个苗头了——根目录一堆散文件、`未命名.md` 这种临时草稿和重要内容混在一起。下面是我的建议，你照着做就行：

### 1. 定死一条规则：收件箱 → 加工 → 入库

```
        ┌─────────────┐
新内容 → │ 00-收件箱    │ ← 所有东西先扔这，不管是课程笔记、灵感、网页剪藏
        └──────┬──────┘
               │ 定期清理
        ┌──────▼──────┐
        │ AI 加工/你自己│ ← 我帮你处理，或者你自己判断
        └──────┬──────┘
               │
    ┌──────────┼──────────┐
    ↓          ↓          ↓
10-知识管理  30-项目管理  40-创作
（高质量）   （工作）     （创意）
```

**现在立刻做的**：把根目录的那些散文件（`未命名.md`、`如何回应.md` 等）要么归档，要么移到对应文件夹。

### 2. 加一个状态标签，区分质量

每篇笔记的 frontmatter 里加一行 `status`：

```yaml
---
status: draft        # 草稿，未加工
status: reviewed     # 已审计，内容可靠
status: final        # 面试可用，金句级
---
```

yaml

这样你用 Dataview 搜 `status: final` 只看高质量内容，搜 `status: draft` 知道哪些还没加工。

### 3. 建一个归档文件夹

```
50-资源库/
  └── 归档/          ← 过时的、不再用的、历史版本的，全扔这
```

不删文件，只是移到这，保持主文件夹干净。

### 4. MOC 只连高质量内容

全库索引（MOC）**只链接 status=reviewed 或 final 的笔记**。垃圾内容不配出现在地图上。这天然形成了一个质量过滤器。

### 5. 命名规范

|类型|命名格式|举例|
|---|---|---|
|核心知识|`领域_主题.md`|`数据治理_数据盘点.md`|
|面试准备|`公司_岗位_用途.md`|`中铝_PMO_面试作战手册.md`|
|临时草稿|`日期_主题_草稿.md`|`2026-06-25_主数据笔记_草稿.md`|
|MOC/索引|`领域_全库索引.md` 或 `_索引.md`|`数据治理_全库索引.md`|

> ⚠️ 绝对禁止 `未命名.md`、`新建文档.md`、`无标题.md`。每个文件名必须一眼看出内容。

## 二、串联模板

下次你要我把知识库里的相关内容串起来，直接复制下面这个，填 `【】` 里的部分：

```markdown
【串联任务】

## 要串什么
【写清楚要串的知识点/主题，可以列多个】
- 例如：数据盘点的二八原则
- 例如：中铝面试里提到的 DCMM

## 串成什么
【选一个】
- [ ] 在现有笔记之间加双向链接就行
- [ ] 写一篇新的串联笔记，把几个知识点拧成一条线
- [ ] 检查这些知识点之间有没有矛盾或重复
- [ ] 出一个面试场景，把这些知识点串成一次回答

## 输出要求
- 放在哪个文件夹：【  】
- 额外约束：【例如：不要自己编案例 / 只基于我已有笔记 / 可以补充行业常识】

## 背景
【可选，告诉我为什么突然要串这些——面试要用？自己梳理？写文档？】
```

markdown

### 举个例子

假设你学了数据质量后，想把散落在不同文件里的相关内容连起来：

```markdown
【串联任务】

## 要串什么
- 数据质量的六维度（在知识体系第八节）
- 及时性稽核的实操（在知识体系第八节）
- 智慧环保项目里的数据质量问题（在30-项目管理/求职/项目逐层深挖）

## 串成什么
- [x] 写一篇新的串联笔记，把几个知识点拧成一条线

## 输出要求
- 放在哪个文件夹：10-知识管理/数据治理课程笔记/
- 额外约束：只基于我已有笔记内容，一条理论配一个项目实例

## 背景
面试要讲项目经验，不想只讲理论
```

## 三、关系图谱过滤规则

> 图谱的价值是展示**知识点之间的关联**，不是把所有文件都扔进去。噪音文件必须排除。

### 图谱中不显示的目录/文件

| 排除路径 | 原因 |
|---|---|
| `50-资源库/归档/` | 过时、历史版本 |
| `50-资源库/模板/` | 格式骨架，不是知识 |
| `50-资源库/图片/` `50-资源库/文章/` `50-资源库/链接/` | 附件、外部采集 |
| `_TODO/` | 临时待办，每日滚动 |
| `00-收件箱/` | 未加工原料 |
| `Excalidraw/` | 绘图文件 |
| `image/` | 图片附件 |

### 如何设置

在 Obsidian **关系图谱** 右上角齿轮 → **Search queries** 填入：

```
-path:50-资源库 -path:_TODO -path:00-收件箱 -path:Excalidraw -path:image
```

再加一道质量锁，只显示 `status: final` 或 `reviewed`：

```
-status:draft
```

### 完整过滤组合（直接粘贴）

```
-path:50-资源库 -path:_TODO -path:00-收件箱 -path:Excalidraw -path:image
-status:draft
```

> 💡 用 `-` 排除，空格分隔多条规则。图谱里出现的全是高质量知识节点和它们之间的连接，一目了然。
