## 盘点

**"做数据治理，我信奉先摸家底再动手——没搞清楚家里有几口锅几个碗，上来就建大数据平台，纯粹是给垃圾堆盖别墅。"**
→ 你怎么看待数据治理 / 你第一步做什么？

**"盘点我有个二八原则：只接 20% 核心业务表，剩下 80% 临时表、备份表、空表，该扔就扔，别给服务器增加减肥负担。"**
→ 怎么确定接哪些表？

**"遇到老系统没文档、字段全是英文缩写？我土办法是反向模拟——让业务点一下按钮，去后台看哪个字段变了，比求甲方翻古董文档快十倍。"**
→ 数据字典缺失怎么处理？

**"做企业项目靠副总裁拍桌子就能推；做政府项目得靠专班+红头文件+三清单，技术只占三分，协调沟通占七分。"**
→ 政府项目有什么特殊性？

**"政务数据治理的钥匙不是技术，是三清单——责任清单定谁出数据，需求清单定谁要数据，负面清单定谁不能碰数据。"**
→ 政务数据共享交换怎么做？

## 接入

**"数据接入就三类活儿：要么搬全家（整库初始化），要么每日补货（单表增量）。补货又分覆盖（直接换新）和追加（老账新账一起存），搞清楚再动手，绝不返工。"**
→ 数据接入有哪些方式？全量和增量怎么选？

## 元数据

**"元数据就是描述数据的数据——好比书的标题、作者、出版社描述书的内容。没有元数据，你看到的只是乱七八糟的数字。"**
→ 什么是元数据？一句话解释。

**"元数据我分三类——技术元数据是字段名表名机器能自动抓的；业务元数据是术语定义指标口径人工填的；管理元数据是权限安全等级管人用的。三管齐下。"**
→ 元数据怎么分类？

**"数据元和元数据很多人搞混。元数据是描述数据的数据，宽泛概念；数据元是描述数据的最小单元标准——比如性别填 1 还是 2，这个取值标准就是数据元定义的。"**
→ 数据元和元数据有什么区别？

**"元数据不只是拿来描述的，它是数据治理的操作系统。做数据地图靠它、做影响分析靠它、做血缘追溯靠它——改一个字段敢不敢改，先查元数据看下游挂了哪些报表。"**
→ 元数据在实际项目中怎么用？

## 数据仓库

**"数据库是菜地，只种一种菜看它现在什么样；数据仓库是地窖，什么菜都往里存，存的是每天摘的，能看出从哪天到哪天都有什么。"**
→ 数据库和数仓有什么区别？

**"数仓四个看家本领——面向主题（分类存放）、集成（多源汇总）、相对稳定（入库不修改）、反映历史变化（所有表带时间戳）。四个缺一不可。"**
→ 数仓有哪些核心特征？

**"没有数仓，做数据分析是空中楼阁。直接查业务库有三座大山——查询慢、上游一变下游崩、历史数据根本查不到。"**
→ 为什么不直接查业务库做分析？

**"数据仓库不是买一个 Oracle 就叫建数仓了。它是完整的方法论——抽取、清洗、分层、建模、主题设计，每个环节都是拼图。只买产品建出来的一定是死仓库。"**
→ 很多人说"我们买了数仓产品"，你怎么看？

## 主数据

**"主数据就一个判断标准——在不在两个以上系统里同时存在？在，就是。CRM 有、ERP 有、财务有、客服也有——一个客户等级变了，四个系统都得跟着变。"**
→ 什么是主数据？怎么判断？

**"主数据是双向的——业务系统改了我得收，我改完了还得发回去；数仓是单向的——只管取数据出分析。一个是业务系统，一个是分析系统，赛道不同。"**
→ 主数据和数仓有什么区别？

**"你在抖音、淘宝、京东、线下门店同时卖货，库存是同一份主数据。一个平台卖出一件，其他平台没实时同步，下一秒就有人下单库存为零——这叫超卖，是事故。"**
→ 为什么主数据要求高实时性？

**"先建主数据，核心数据已经干干净净了，数仓省一半力气；先建数仓，从里面把主数据识别出来同步做初始化。两条路互相兜底。"**
→ 主数据和数仓先建哪个？

## 数据质量

**"离线数据加工就像凌晨的流水线——12 点半开始，第二天上班前必须跑完。每个环节设时间阈值，超时就报警。不等第二天早上发现指标没出来再追责。"**
→ 数据质量怎么保证及时性？

**"拉链表就是数仓里记历史账的手段——业务系统更新直接覆盖，但我不能丢历史，所以把旧记录关了再开新记录，用 start_date 和 end_date 管着。"**
→ 什么是拉链表？为什么用？

**"拉链表最常见的三个毛病：断链是时间有空档，交叉链是两条记录打架，重复链是同一时间多条相同记录。每天跑稽核脚本扫一遍。"**
→ 拉链表的稽核主要查什么？

## 数据安全

**"以前数据安全是锦上添花，现在是法律强制。21 年数据安全法和个保法一出，采集了数据不上加密脱敏就是违法——Facebook 被罚 20 亿就是前车之鉴。"**
→ 数据安全现在是什么地位？

**"数据安全我理解就六个字——合法、正当、必要。合法是有法律依据，正当是目的要讲清楚，必要是只拿最少的数据，不多要一寸。"**
→ 数据安全的基本原则是什么？

**"对称加密是同一把钥匙开门锁门，快但钥匙怎么安全给对方是个问题；非对称是锁（公钥）随便给，钥匙（私钥）自己拿着。政府项目还得用国密，这是硬性要求。"**
→ 数据加密有哪些方式？

**"等保评整体——机房+网络+硬件+软件+制度全包；DSMM 只评数据安全的成熟度，从采集到销毁全生命周期。两个都是五级，考的不是同一张卷子。"**
→ 等保和 DSMM 有什么区别？

**"数据安全不是装个防火墙就完事了。要从五个层面搭：先定原则和等级，再在采集传输存储处理交换每个环节加密脱敏权限审计四件套，最后组织和运维兜底。缺一层都有漏洞。"**
→ 数据安全体系怎么搭建？

## 政务数据服务

**"政务数据共享就四个字——编目、挂载、级联、交换。先编好目录让人找得到，再挂到后面真实数据上让人拿得到，然后上下级平台同步起来让跨地域也能访问，最后走共享交换平台把数据搬过去。"**
→ 政务数据共享的整体流程是怎样的？

**"政务体系里有两个网站——共享网站是内部用的，各厅局之间调数据；开放网站是对老百姓的，你能在上面看到政府公开了哪些数据。一个在内网一个在公网，服务对象完全不一样。"**
→ 政务数据共享网站和开放网站有什么区别？

**"有条件共享和无条件共享就差一个审核环节。无条件的数据申请就给你配交换任务，有条件的数据提供方要先看你的理由充不充分，不充分就打回去。"**
→ 政务共享数据有条件共享和无条件共享的区别是什么？

**"我在项目上做过完整的数据服务实施——目录设计是咨询人员干的，资源挂载是数据技术干的，共享交换是各委办局的人员日常操作的，最后数据应用是业务人员和社会公众用的。四波人四件事，缺一波都跑不通。"**
