# 数据治理 · 记忆闪卡大全

> 一问一答，口语化。按主题索引，地铁上或睡前刷一遍。

---

## 盘点

**Q: 为什么要先做数据盘点？**
A: 搬家前先扔破烂。不盘点直接接数据，新平台（新家）会被垃圾表（发霉的衣服、报废电器）塞满。

**Q: 数据盘点三大目的是什么？**
A: ① 摸家底（搞清楚有啥）② 为建模做准备（避免返工）③ 摸清业务真实需求（减少甲方瞎改）

**Q: 没有数据字典怎么办？**
A: 反向模拟法——让业务在系统里做一次操作，去后台看哪个表哪个字段变了，反向推出字段含义。

**Q: 企业 vs 政府做盘点最大区别？**
A: 企业有高层牵头就行；政府得搞专班+红头文件+三清单，技术占三分协调占七分。

**Q: 三清单是哪三张？**
A: 责任清单（谁出数据）、需求清单（谁要数据）、负面清单（谁不能碰）。

**Q: 表级分析的二八原则什么意思？**
A: 一百张表只接 20 张核心的。临时表、备份表、空表、测试数据全过滤。

## 接入

**Q: 数据接入有哪几种数据类型？**
A: 结构化（关系库）、半结构化（JSON/MongoDB）、非结构化（图片视频）、实时流（CDC/消息队列）。

**Q: 三种接入方式分别在什么时候用？**
A: 整库用于初始化（一次性搬历史）；单表用于日常增量；SQL 用于特定筛选（用得少）。

**Q: 全量覆盖和全量追加区别？**
A: 覆盖→每天换新的不存历史（代码表）；追加→存一份新的同时保留旧的（核心业务表，要历史快照）。

**Q: 增量接入用在什么场景？**
A: 历史不会变的表，如日志表、流水表。每天只拿新增的，不用全量拉。

**Q: Sqoop 接入走哪四步？**
A: 取元数据 → Hive 建表指定 HDFS 路径 → 抽数据到临时目录 → Load 到目标路径。

## 元数据

**Q: 什么是元数据？**
A: 描述数据的数据。好比书名、作者描述书的内容——书是数据，书名作者是元数据。

**Q: 没有元数据会怎样？**
A: 你看到一堆 1、2、20、10、40，根本不知道这些数字是年龄还是价格。

**Q: 元数据分哪三类？**
A: 技术元数据（字段名表名，机器自动抓）；业务元数据（术语定义指标口径，人工完善）；管理元数据（权限安全等级，管理维度）。

**Q: 数据元和元数据有什么区别？**
A: 元数据是描述数据的数据（宽泛）；数据元是最小单元标准（如规定性别字段 1=男 2=女）。

**Q: 元数据能拿来干什么？**
A: 做数据地图（搜索发现数据）、做影响分析（改字段查下游影响）、做血缘分析（数据从哪来去哪了）。

## 数据仓库

**Q: 数据库和数仓最本质区别？**
A: 数据库存当前时点（记录被覆盖），数仓存历史变化（带时间戳，只追加不覆盖）。

**Q: 数仓是一个具体产品吗？**
A: 不是。数据库 Oracle/MySQL 是产品；数仓是方法论+建设过程——抽取、清洗、分层、建模。

**Q: 数仓四大特性？**
A: ① 面向主题 ② 集成 ③ 相对稳定 ④ 反映历史变化。

**Q: 为什么不直接查业务系统做分析？**
A: 三个问题——海量表多表关联慢、上游字段变化下游崩、查不到历史数据。

**Q: 数仓三层分层？**
A: ODS（贴源层）→ DW（汇总层）→ DM（数据集市层）。

**Q: 如果有数仓，怎么查去年的指标？**
A: 进 DW/DM 层按时间戳筛选——业务系统做不到。

## 主数据

**Q: 什么数据叫主数据？**
A: 同时存在于两个以上系统中，被多部门共享引用的数据。如客户信息、产品信息。

**Q: MDM 核心目的？**
A: 解决不同系统间数据一致性问题——一个系统改了，MDM 统一同步到所有系统。

**Q: 主数据和数仓最大区别？**
A: 主数据双向（收变更+发同步）、实时性极高；数仓单向（只取数据出分析）、一般 T+1。

**Q: 为什么库存要求高实时性？**
A: 多平台共享一份库存，一个平台卖出必须实时同步，否则超卖是事故。

**Q: 主数据和数仓怎么配合？**
A: 先建 MDM→核心数据干净了数仓省力；先建数仓→从数仓识别主数据做 MDM 初始化。互相兜底。

## 数据质量

**Q: 及时性稽核是干什么的？**
A: 检查每个加工环节的时间是否符合预期，超时就报警。不等第二天发现指标没了再追责。

**Q: 及时性稽核监控哪几个环节？**
A: ODS 完成时间、DW 加工时间、整个工作流结束时间。报警前置，半夜发现问题半夜修。

**Q: 什么是拉链表？**
A: 数仓记录历史变化的手段——start_date（开始生效）+ end_date（关闭日期），关闭旧记录插入新记录。

**Q: 怎么查拉链表中某一天的数据？**
A: WHERE start_date <= 查询日期 AND end_date >= 查询日期。

**Q: 拉链表三种异常？**
A: 断链（时间有空档）、交叉链（时间重叠）、重复链（同一时间多条相同）。

**Q: 为什么不能像业务系统那样直接覆盖？**
A: 业务系统只要当前状态，数仓要为历史分析负责。去年这时用户的手机号，业务表已查不到了。

## 数据安全

**Q: 跟数据安全相关的国家法律有哪些？**
A: 国家安全法（2015）→ 网络安全法（2017）→ 数据安全法（2021）→ 个人信息保护法（2021）。

**Q: 等保 2.0 比 1.0 新增了什么？**
A: 五个扩展——云计算、移动互联网、物联网、工控系统、大数据安全。政府项目一般要求等保三级。

**Q: 等保三级评什么？**
A: 评整个组织——机房、网络、硬件、制度体系、软件全涵盖，不是你开发的软件单独过。

**Q: DSMM 是什么？**
A: 数据安全能力成熟度模型。围绕数据全生命周期，从组织、制度、技术、人员四个维度评 1-5 级。

**Q: 等保和 DSMM 不同？**
A: 等保评整体（机房+网络+硬件+软件+制度），DSMM 只评数据安全。两个都是五级。

**Q: 数据安全六大原则？**
A: 合法正当、目的明确、最小够用、全程可控、动态控制、权责一致。

**Q: 对称加密 vs 非对称加密？**
A: 对称→同密钥加解密（快，密钥分发难）；非对称→公钥加密私钥解密（安全，慢）。

**Q: 为什么政府项目强调国密算法？**
A: 国家密码局认定的国产算法（SM2/SM3/SM4），政府、央企强制要求。

**Q: 加密和脱敏有什么区别？**
A: 加密可逆（有密钥能解开）；脱敏不可逆（138****0000 拿到也无法还原）。

**Q: 山东那家公司为什么被罚？**
A: 采集个人信息放公有云但没做任何安全防护（没加密没脱敏）→ 即使没售卖也违法。

## 政务数据服务

**Q: 政务数据共享整体流程？**
A: 四步——编目（建目录）→ 级联（上下级同步）→ 挂载（目录连数据）→ 申请使用（检索→申请→审核→共享交换）。

**Q: 共享网站和开放网站有什么区别？**
A: 共享网站=政务内网，各厅局内部用，互联网访问不到；开放网站=对老百姓公开，任何人都能查看申请。

**Q: 资源目录系统是干什么的？**
A: 相当于图书馆的书目检索系统——对政务数据编目、注册、变更、申请，让使用者能搜索和发现数据。

**Q: 共享交换平台是干什么的？**
A: 真正的搬运系统——审核通过后，把数据从提供方前置机搬到需求方前置机，支持库表、接口、文件三种方式。

**Q: 有条件共享和无条件共享差在哪？**
A: 无条件→申请后直接配交换任务；有条件→多一个审核环节，提供方判断申请理由是否充分。

**Q: 上下级平台级联是什么？**
A: 省级目录上报到国家，市级上报到省级。解决跨地域数据共享——比如山西省访问山东省的数据，通过国家级目录和平台。

**Q: 单次使用和多次使用有什么区别？**
A: 单次=一次性下载拿走；多次=配定期调度任务（如每天凌晨同步增量），适用于需要持续更新的结构化库表。

**Q: 数据获取有哪三种方式？**
A: 库表（直连数据库）、接口（API 调用）、文件（文件传输）。

**Q: 为什么政务数据必须走共享交换平台，不能私下传？**
A: 安全可控——有申请记录、审核流程、审计日志，确保数据使用合规、目的明确、最小够用。

**Q: 三清单和数据编目什么关系？**
A: 三清单是编目的前置输入——责任清单决定目录有什么，需求清单决定优先编什么，负面清单决定什么不能进目录。

## 企业数据服务

**Q: 企业数据获取有哪三种方式？**
A: JDBC 直连（简单但安全性低）、API 接口（安全但不适合大数据量）、数据同步（适合批量定期推送）。

**Q: JDBC 直连的优缺点？**
A: 优点→最简单，直接连库；缺点→暴露数据库连接信息，安全性低，不适合跨部门。

**Q: API 方式的优缺点？**
A: 优点→不暴露后台存储计算，更安全；缺点→数据量大时性能差。

**Q: 数据同步的优缺点？**
A: 优点→适合大批量、周期性推送；缺点→需要额外的同步引擎和调度系统。

**Q: 为什么企业也需要数据资产目录？**
A: 传统模式下数据对业务方是黑盒，不知道有什么数据可用。目录让业务方先看得见，再按需申请。

**Q: 数据同步的三种模式？**
A: 库→库（直连同步，简单）、库→文件→需求方（FTP 中转，安全隔离好）、文件→文件（两个 FTP 跨网桥接，适用跨网段）。

**Q: 库→库同步什么场景用？**
A: 上层应用少、非跨部门，简单直接，但双方数据库信息互相暴露，有安全风险。

**Q: 文件→文件同步什么场景用？**
A: 跨网段场景——集团和子公司、子公司之间网络隔离，通过两个跨双网的 FTP 服务器接驳。

**Q: 同步任务配置流程？**
A: 创建数据源 → 选表 → 字段映射 → 执行测试 → 挂定时调度 → 任务监控+异常报警。

**Q: 数据同步和数据接入什么关系？**
A: 方向相反——接入是从业务库拉到大数据平台，同步是从大数据平台推回给业务方。技术手段一样。

**Q: 为什么 Hive 上直接生成 API 不行？**
A: Hive 读取效率慢。标准做法是 Hive 上面加 Presto（内存分布式查询引擎），通过 Presto 对外暴露 API。

**Q: API 类资产和库表类资产有什么区别？**
A: API 类=通过配置把数据生成 API 接口，不暴露底层存储；库表类=直接开放物理表访问。两种都挂在数据资产目录下。
