# 数据治理 · 主数据管理

## ✨ 这是啥

> **主数据 = 同时存在于两个或两个以上业务系统中的共享数据**，比如客户信息、产品信息、人员信息。一个系统改了，其他系统也得跟着改——主数据管理系统（MDM）就是在中间做这个统一协调的角色。

主数据在企业和政务中都极其核心，也极其复杂——因为它跟不同的系统、部门、业务几乎都有关系。

---

## 🎯 有啥用

### 没有主数据管理会怎样？

企业内部"烟囱式"建了很多系统：CRM、ERP、客服、财务……每个系统都各自维护一份客户数据。一个客户的等级从银牌变了金牌，CRM 改了，但 ERP 和财务没改——**同一个客户在不同系统里数据不一致**。

主数据管理系统的核心目的：**解决不同系统间的数据一致性问题。**

### 主数据管理系统的运作方式

```
CRM 变更客户等级 → 通知主数据管理系统 → MDM 统一分发 → ERP/客服/财务同步更新
```

> 关键：主数据管理系统是**双向的**——从业务系统接收变更，再同步回其他业务系统。

---

## 🛠️ 怎么用

### 一、主数据的定义门槛

一个数据要被称为主数据，必须满足：

| 条件 | 说明 |
|---|---|
| **跨系统存在** | 至少存在于两个或以上业务系统 |
| **共享使用** | 被多个部门/业务共同引用 |
| **需要一致** | 一个地方变了，其他地方必须同步 |

典型主数据举例：
- **企业**：客户、产品、供应商、组织架构、人员
- **政务**：人口、法人、地理信息（公安/民政/税务/法院等几十个委办局都在用）

### 二、主数据管理系统的核心流程

```
识别主数据 → 集成汇聚 → 建立标准 → 质量稽核 → 清洗转换 → 统一建模 → 分发同步
```

看起来眼熟？流程确实和数据仓库很像——都要做盘点、接入、标准制定、质量稽核、清洗转换、数据建模。

### 三、主数据 vs 数据仓库（关键区分）

| 维度 | 主数据管理系统（MDM） | 数据仓库 |
|---|---|---|
| **定位** | 业务系统（支撑业务流程） | 分析系统（支撑分析决策） |
| **数据流向** | **双向**：接收变更 → 分发到其他系统 | **单向**：从业务系统取数 → 只出分析结果 |
| **数据范围** | 只管理**跨系统的核心共享数据**（子集） | 管理**全面数据**（全集） |
| **实时性** | **极高**（库存超卖就是事故） | **相对低**（多数 T+1 就够） |
| **最终价值** | 数据一致性、协同、共享 | 历史分析、决策支撑 |

> **经典场景**：你在抖音、淘宝、京东同时卖货，线下还有门店——库存是同一份主数据。一个渠道卖出一件，所有渠道看到的库存必须**实时**同步减少，否则就是超卖事故。这靠的不是数据仓库，是靠主数据管理系统。

### 四、主数据和数据仓库可以互相补充

- **先建主数据 → 后建数仓**：主数据库里已经有了最核心的共享数据，可以作为数仓的一个高质量数据来源
- **先建数仓 → 后建主数据**：从数仓里识别出主数据，同步到主数据库做初始化，再基于此做主数据的运营管理
