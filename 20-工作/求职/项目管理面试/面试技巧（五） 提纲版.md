## 一、自我介绍两种策略

### 常规路线（推荐）

```

我叫 XXX，工作 11 年，4 家公司。

逐家介绍：时间 + 公司名 + 岗位 + 项目 1、2（每项目 ≤20 秒/70-80 字）

```
**原则：** 留充足时间让面试官通过提问了解你。
### 进阶路线（有条件才用）

- 前提：自我总结能力好 + 非常自信 + 明确知道面试官想要什么

- 不满足前提走常规路线
---
## 二、用 AI 吃透项目（六步法）

> 简历 = 课本，AI = 帮你深挖课本里知识的工具
### 基础六问（依次问 AI）

1. "基于这段项目介绍，梳理 200 字面试用项目介绍"

2. "梳理出这个项目的项目目标"

3. "梳理出这个项目的需求收集及分析"

4. 针对细节深挖："IoT 设备传输什么数据？"

5. "描述这个项目的解决方案"

6. "总结项目交付的核心要点"
### 有完整文档时

- 拖进 AI → "总结 200 字项目介绍"

- "分析核心功能/核心点"

- "从 XX 角度讲述这个项目，给我提示"
### 关键动作

- 花一天时间把那几个项目全部吃透

- 不停深问 AI，不要只问一次

- 最终用**自己的话**总结，不要照着 AI 读
---
## 三、三分钟六环节（700 字，总分方式）

```

项目内容→需求分析→确定目标→解决方案→项目管理→交付核心

```

| 环节 | 内容 | 时长参考 |

|------|------|---------|

| ① 项目内容 | 项目是什么，一句话 | 20 秒 |

| ② 需求分析 | 调研方 + 需求，一两句话 | 30 秒 |

| ③ 项目目标 | 1、2、3、4…… | 20 秒 |

| ④ 执行路径 | 总周期 + N 个交付里程碑 | 30 秒 |

| ⑤ 解决方案 | 技术架构（30-40 字） + 产品核心能力（3-4 个） | 40 秒 |

| ⑥ 交付 | 验收管理（协同谁）+ 落地 | 30 秒 |

- 语速：220-235 字/分钟

- **不能用书面语言**，反复练习
---
## 四、风险管理：化整为零 + 问题前置

### 核心逻辑

> 拆里程碑 → 管好每个里程碑的风险 = 管好整个项目的风险

面试时**带入场景**，不要列概念。
### 问题前置（不是解决方案）

**示例：团队 10 人缺 2 人，怎么办？**

- ❌ 讲解决方案

- ✅ 讲前置预案：

- 预案一：提前跟 CTO 确认——前后端各谁能顶上

- 预案二：3%-5% 合同额备用金 → 找外包

> **风险从来不是"管理"出来的，是"前置"出来的。项目出现风险 = 工作已有失职迹象。**

---
## 五、变更问题：建立规则，不是说服

### 三个"不是"

- 不是说服领导接受客户白嫖

- 不是说走服客户放弃变更

- **不是走变更管理流程**（如果所有问题都走流程，要项目经理何用？） 

### 核心答案：规则前置

- 在项目之初反复强调职责和权限边界

- "字有问题我分分钟改，框架要动我做不了主，找我领导去"

- 先小人后君子，不要怕对方烦

- 不把 PMP 书本搬进面试

---
## 六、"最难项目"标准答案
### 不能说

- "部门多难沟通" → 那是你沟通能力不行

- "涉及全国 100+ 单位 400+ 部门" → 那是时间问题
### 标准答案：资源不够用

```

缺钱 → 利润低（40% → 10%）

缺人 → 公司没有多余人力

缺时间 → 客户压缩交付周期

```
### 解决：精细化管理

1. **重新拆解需求** → 哪些精简？哪些伪需求？哪些放二期？

2. **重新规划排期** → 研发评估习惯性留 20%-30% 余量，压缩掉

**核心价值：** 把需求吃透，才有跟客户和团队平等对话的能力。只拉站会、做会议纪要的项目经理没有存在价值。

## 七、项目交付的双重视角

| | 对内交付 | 对外交付 |

|------|---------|---------|

| 关注方 | 领导/公司 | 客户 |

| 关注点 | 成本、风险、范围 | 进度、质量 |

---
## 八、客情管理四大支柱

| 支柱 | 操作 |

|------|------|

| 汇报节点 | 一周或两周一次，不超过两周；领导没空也不能跳过 |

| 链接频率 | 早请示晚汇报，日常聊几句 |

| 规则前置 | 先小人后君子，反复强调，说烦了也说 |

| 专业能力 | 行业理解 + 需求深入 → 平等对话；技术难题找支援但要自己能说一二三 |


---
## 九、范围管理：还是落到里程碑 

> "第 2 个里程碑（原型阶段），控制产品经理按 12 项功能做，不遗漏不添油加醋。每个里程碑管好范围 = 整个项目管好范围。" 

面试官是 HR → 简略说；是项目经理/总监 → 带场景细说。

## 十、总览

| 环节 | 心法 |

|------|------|

| 自我介绍 | 常规路线为主，每个项目 20 秒 |

| 项目准备 | AI 六步深挖，花一天吃透项目 |

| 3 分钟介绍 | 六环节 700 字，口语化，总分式 |

| 风险/变更 | 化整为零，问题前置，规则前置 |

| 最难项目 | 资源不够用 → 精细化拆需求+压排期 |

| 客情管理 | 汇报 + 链接 + 规则 + 专业，四管齐下 |
