## 一、自我介绍参数

| 项目 | 标准 |

|------|------|

| 语速 | 约 235 字/分钟 |

| 字数 | 800-1400 字 |

| 时长 | 4-8 分钟（8 分钟 ≈ 1700 字） |

| 核心目标 | 我经历了什么？我能做什么？我做了什么？ |

---
## 二、自我介绍模板

```

模板结构：

1. 基本信息（姓名、学历，专升本不提及毕业院校）

2. 工作年限总述

3. 按公司逐段展开：

- 公司 + 时间 + 主营

- 项目 1、2、3（名字+类型+担任角色）

- 角色固定：项目经理和解决方案专家

4. 收尾总结（300 字以内）

```

### 为什么用"解决方案专家"标签

- 不是职位，是能力视角

- 体现从 0 到 1 的全面把控力

- 比"我管过项目"更立体
### 必须加收尾总结

- 5-8 分钟里面试官注意力会分散

- 最后一段帮他重新聚焦："他做过什么、能做什么"

- 例："10 年间我从技术转型项目经理，经历了 XX、XX 等项目"

## 三、项目介绍规则

| 规则 | 标准 |

|------|------|

| 字数 | 300 字 |

| 时长 | 1 分 30 秒 |

| 背景 | ❌ 不介绍（一句话带过即可） |
### 介绍结构

```

1. 谁要做一件什么事？目标是什么？

2. 需求分析：调研对象 + 具体需求痛点（颗粒度到业务领域即可）

3. 方案设计：你做了什么对接/规划（非技术实现）

4. 项目组建：甲方 + 乙方 + 关键第三方

```

### 参考话术（以资金监管系统为例）

- 需求：明确了 XX 在预售资金监管、销售收款、财务入账及凭证生成等方面的需求与痛点

- 方案：设计收单平台与监管系统的对接方案，明确 API 协议、接口规范、安全机制、成本分摊

## 四、TO B vs TO G 沟通

| | TO B（企业） | TO G（政府） |

|------|------------|------------|

| 沟通难度 | 相对便利 | 需特别注意 |

| 核心原则 | 行业相通，交流顺畅 | **三不谈：** 不谈数据细节、不谈技术细节、不谈产品细节 |

| 聊什么 | 需求、痛点、方案 | **实现后的效果，能为他们带来什么** |

### TO G 加分项：知道流程节点

政府立项必经：支委会 → 党委会 → 经理办公会 → 专家研讨会 → 各部门交叉会议

面试亮点：**我能为每个节点制作对应的汇报材料。**

## 五、项目管理管控要点

### 四大核心

范围管理 + 进度管理 + 质量管理 + 风险管理
### 成本管理的特殊性

- 只说"人、采购、时间"三件事不超标

- 实际中成本管理容易被财务和领导干预（税务原因），面试中不必深究
### 偏差应对

- 进度偏差 → 动用 3%-5% 备用金 → 找外包 / 自由工作者

## 六、投递招呼语："前 14 字"规则

> HR 只看到招呼语的前 14 个字 → 必须包含你的行业或项目类型
### 三个对标顺序

1. 行业对标（金融/健康/制造）

2. 项目类型对标（信贷/业财/智慧园区）

3. 职责对标（PM/交付/产品）
### 操作方式

- 准备 5-6 个招呼语模板

- 针对不同岗位动态调整前 14 字

- "您好，我精通**融资租赁**、健康领域……"

- "您好，我深耕**金融**、信贷领域……"

## 七、搜索与投递节奏
### 搜索逻辑

```

先总结你做过的项目类型 → 用类型关键词搜索 → 轮换关键词 → 搜完再泛投

```

| 做过 | 搜索词 |

|------|--------|

| 资金监管/财务共享 | 业财一体 |

| 资管系统 | 资管 |

| 租赁系统 | 租赁平台 / 融资租赁系统 |

| 数字化 | 数字化产品 / 数字化项目 |
### 投递节奏

- 9 点、12 点、下午分批搜（一个岗位日曝光约 500 次）

- 本平台搜无可搜 → 转智联、前程无忧

## 八、投了不回的原因

| 原因 | 说明 |

|------|------|

| 已招到 | 岗位未下架 |

## 九、日常训练

每天 15 分钟 × 7 天，对着镜子讲自我介绍。

**三件要掌握的事：**

1. 自我介绍怎么讲

2. 项目怎么介绍

3. 怎么投、怎么调招呼语、怎么动态搜索
